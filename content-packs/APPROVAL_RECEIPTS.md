# Reviewed Pack Approval Receipts

PorchQuest369 reviewed packs are kept separate from app code. A pack is promoted only after it passes the contributor-safe review loop.

## Receipt Format

Each reviewed pack should be able to produce a receipt with:

- pack ID and title
- schema validation status
- promotion checklist status
- playtest transcript export
- maintainer approval note
- safe-content boundary note

## Safe Content Boundary

Reviewed route packs should stay in cozy fantasy/adventure territory. They should not include secrets, private user data, API keys, real-world targeting instructions, medical/legal claims, or content that bypasses the claim-safe design of PorchQuest369.
